Проверено:

- Subscription webhook проверяет HMAC и допустимый возраст timestamp.
- API key показывается целиком только при создании; в D1 сохраняются hash и prefix; revoke и create пишутся в audit.
- В `wrangler.jsonc` production secrets не обнаружены; быстрый secret scan не нашёл очевидных приватных ключей и известных token prefixes.

Блокирующие замечания:

1. **P0 - захват аккаунта через неподтверждённый email.** `/api/register` ищет существующего пользователя по переданному клиентом email и сразу создаёт для него authenticated session. Подтверждение владения email отсутствует. Злоумышленник, знающий email пользователя, может получить сессию его аккаунта. До исправления нельзя считать session-derived identity безопасной.
2. **P0 - Google OAuth отсутствует.** Нет Authorization Code + PKCE, server-side `state`/`nonce`, проверки issuer/audience/expiry, `email_verified` и redirect allowlist.
3. **P0 - admin hardening не выполнен.** `requireAdmin` всё ещё принимает `token` и `adminUser` из URL query. Admin token хранится браузером в `sessionStorage`; нет коротких admin sessions, MFA/compensating control и RBAC. IP rate limit `120/min` не заменяет failed-login throttling.
4. **P0 - subscription webhook не идемпотентен.** Provider event id не принимается и не сохраняется; повтор валидно подписанного события создаёт новую запись. Нет проверки provider account/environment и разрешённых переходов статуса.
5. **P0 - API token enforcement не реализован.** Созданные API keys не аутентифицируют публичный analysis/scanner endpoint; scopes задаются клиентом без server-side allowlist; нет scope enforcement, per-token limits и abuse detection. Hash ключа не усилен server-side pepper/keyed hash.
6. **P1 - internal auth допускает обход HMAC через общий Bearer secret.** Для Bearer-запроса не проверяются method/path/body digest. Нет обязательного `X-Request-Id`, replay storage и key id для ротации; подписанный запрос можно повторить в пределах timestamp window.
7. **P1 - CSRF защита неполная.** Synchronizer token проверяется, но отсутствует обязательная same-origin проверка `Origin`/`Referer` для cookie-authenticated write requests.
8. **P1 - session lifecycle не соответствует критериям.** Установлен 30-дневный absolute TTL, нет 30-минутного idle timeout, logout/revocation flow и production `__Host-` cookie prefix.
