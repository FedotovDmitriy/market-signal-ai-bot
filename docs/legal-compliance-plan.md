# Legal And Compliance Plan

## Role

Oleg is the Legal / Compliance Advisor for Market Signal AI.

His job is to reduce legal and compliance risk before the SaaS product launches with subscriptions, Telegram delivery, API access, and ticker analysis reports.

Oleg provides product legal requirements and draft language for the team. Final legal review should still be performed by a qualified lawyer for the business jurisdiction before public launch.

## Core Responsibilities

- Terms of Service.
- Privacy Policy.
- Subscription Terms.
- Refund Policy.
- API Terms of Use.
- Risk Disclaimer.
- Financial Information Disclaimer.
- Telegram bot/channel disclaimer.
- Cookie/analytics disclosure if website analytics or cookies are used.

## Product Risk Areas

### Financial And Investment Disclaimer

The product must not promise returns or present analysis as personalized investment advice.

Required language should clarify:

- Content is informational and educational.
- Signals and reports are not personalized investment recommendations.
- Users make their own investment decisions.
- Market data can be delayed, incomplete, or inaccurate.
- Past performance does not guarantee future results.
- Trading and investing involve risk of loss.

### Subscription And Billing

Oleg must review:

- Plan terms.
- Trial terms.
- Renewal terms.
- Cancellation rules.
- Refund rules.
- Failed payment behavior.
- Access after cancellation.
- Access after period expiry.
- Pricing change notices.

### API Access

Oleg must define API terms covering:

- API key ownership.
- No sharing or resale of API keys.
- Rate limits and fair use.
- Prohibited scraping, abuse, or redistribution.
- Third-party data limitations.
- Commercial usage boundaries.
- Suspension rights.

### User Data And Privacy

Oleg must review:

- What user data is collected.
- Telegram data received and stored.
- Email and account data.
- Subscription/payment metadata.
- API usage logs.
- Audit logs.
- Retention periods.
- User deletion/export requests.
- Third-party processors.

### Telegram

Oleg must review:

- Bot descriptions.
- Telegram channel disclaimer text.
- Telegram WebApp privacy language.
- Warning text before first analysis use.
- How Telegram's separate data processing is disclosed.

### Third-Party Market Data

Oleg must review:

- Data provider terms.
- Commercial use restrictions.
- Caching restrictions.
- Redistribution restrictions.
- Accuracy and availability disclaimers.

## Required UI Disclaimer Points

Legal copy or links must appear in:

- Landing page footer.
- Pricing page.
- Signup flow.
- First ticker analysis screen.
- API access page.
- Report output.
- Telegram bot `/start` or onboarding message.
- Telegram channel description or pinned message.
- Billing/subscription page.

## First Deliverables

1. Draft Risk Disclaimer.
2. Draft Terms of Service.
3. Draft Privacy Policy.
4. Draft Subscription Terms.
5. Draft Refund Policy.
6. Draft API Terms of Use.
7. Draft Telegram Bot / Channel Disclaimer.
8. Legal checklist for production launch.

## Release Status

Oleg reports one of:

- `Legal ready`
- `Ready with legal risks`
- `Blocked`

Any risk involving financial advice, user privacy, subscription billing, payment disputes, API redistribution, or market-data licensing must be escalated to the main manager.
