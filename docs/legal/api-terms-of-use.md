# API Terms Of Use

Status: working product draft, not final legal advice.
Last updated: 2026-06-18.

These API Terms apply to use of Market Signal AI APIs, API keys, webhooks, internal or external endpoints, analysis request endpoints, and related developer features.

## 1. API License

Subject to these API Terms, the Terms of Service, payment obligations, and plan limits, Market Signal AI grants the user a limited, revocable, non-exclusive, non-transferable right to access the API for permitted internal business or personal research use.

## 2. API Keys

API keys are confidential credentials. Users must:

- keep API keys secret;
- store API keys securely;
- rotate compromised keys immediately;
- not share keys with third parties;
- not embed keys in public code, public websites, mobile apps, logs, screenshots, or client-side bundles;
- promptly revoke keys that are no longer needed.

Users are responsible for all API activity under their keys.

## 3. Permitted Use

Unless a written enterprise agreement says otherwise, API use is limited to:

- requesting ticker analysis for the user's own research;
- integrating analysis into internal dashboards;
- retrieving the user's own account, subscription, usage, and report history;
- building internal workflows that respect plan limits and disclaimers.

## 4. Restricted Use

Users must not:

- resell, sublicense, rent, lease, publish, or redistribute API data or reports as a standalone product;
- use API outputs as personalized investment advice for third parties;
- claim Market Signal AI endorses a trading decision;
- remove or hide required disclaimers;
- scrape, crawl, benchmark abusively, or bypass rate limits;
- use multiple accounts or keys to evade limits;
- reverse engineer the API or model behavior;
- overload infrastructure or perform denial-of-service activity;
- use the API for unlawful market manipulation, fraud, spam, phishing, or misleading promotions;
- store or cache third-party market data beyond permitted license limits;
- submit sensitive personal data, trading account credentials, private keys, or payment card data.

## 5. Commercial Use

Commercial redistribution, white-label products, public apps, paid newsletters, trading communities, managed account services, broker integrations, or products that expose Market Signal AI outputs to third parties require written approval and may require a separate enterprise agreement.

## 6. Rate Limits And Usage Limits

Market Signal AI may enforce rate limits, quota limits, request size limits, concurrency limits, endpoint limits, and abuse controls. Limits may vary by plan.

The service may throttle, reject, delay, suspend, or terminate API access if usage creates operational, legal, security, data licensing, or abuse risk.

## 7. API Outputs

API responses may include market analysis, ticker matches, confidence scores, signal labels, risk notes, summaries, and reports. These outputs are informational only and are not investment advice or trading recommendations.

Users must independently verify API outputs before relying on them.

## 8. Data And Caching

Users may cache API responses only as allowed by their plan, these API Terms, and third-party data license restrictions.

Market Signal AI may restrict caching, storage, redistribution, display, or downstream use of certain market data or news-derived content.

## 9. Attribution And Disclaimers

Public or third-party-facing use requires prior written approval and must include attribution and disclaimers approved by Market Signal AI.

Minimum disclaimer:

> Market Signal AI provides informational market analysis only. It is not investment advice and does not recommend buying or selling any financial instrument.

## 10. Monitoring

Market Signal AI may monitor API usage for billing, rate limits, abuse prevention, security, service improvement, and compliance.

## 11. Changes

APIs may change over time. Market Signal AI should provide reasonable notice for breaking changes where commercially practical, but may make immediate changes for security, legal, provider, or abuse reasons.

## 12. Suspension And Termination

Market Signal AI may suspend or terminate API access for non-payment, abuse, legal risk, security risk, excessive errors, data license violations, key sharing, or breach of these API Terms.

Local legal review required before launch:

- market data provider redistribution and caching restrictions;
- public API commercial use rules;
- securities/investment-advice perimeter for third-party-facing integrations;
- export control and sanctions restrictions if applicable;
- enterprise contract terms for B2B API customers.
